Choosing an Appropriate IDE

What is an IDE?

IDE stands for Integrated Development Environment. It is a software application that combines all of the features and tools needed by a software developer.

Examples:
  • Text Editor
  • Project Editor
  • Tool bar
  • Output Viewer

An IDE consists of:

Source code editor: text editor that assists in writing software code with syntax highlighting, language specific auto-completion. and checking for bugs as code being written.

Local build automation: utilities that can automate simple and repeatable tasks as part of creating a local build of software such as compiling computer source code, packaging binary code, and running automated tests.

Debugger: program that is for testing other programs that graphically display the location of a bug in the developers code.

Popular Kinds of IDEs

The number of supported languages: Some IDEs are dedicated to one language which makes them a better match for a specific programming paradigm.(IntelliJ). Other IDEs have a broad array of supported languages all in one. (Visual Studio)

Supported Operating System(s): A user's operating system will constrain which IDEs are viable, and if the application being developed is intended for an end user with a specific operating system, this may be an additional constraint.

Automation features: Most IDEs include the 3 key features of a text editor, build automation, and debugger, many include support for additional features such as refactoring, code search, and Continuous integration and continuous deployment (CI/CD) tools.

Impact on system performance: IDE's memory footprint may be important to consider if a developer wants to run other memory-intensive applications concurrently.

Plugins and extensions: IDEs include ability to customize workflow to match a developer's needs and preferences.
